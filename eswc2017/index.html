Supporting Ontology Engineering with OnToology

ESWC 2017
Authors: Ahmad Alobaid, Daniel Garijo, Maria Poveda-Villalon, Idafen Santana-Perez, Alba Fernandez, and Oscar Corcho

Abstract

Due to the increasing uptake of semantic technologies, ontologies have started to be adopted in many software development projects. As a result, several challenges arise for those teams that have to deal with the interaction between usual ontology engineering activities and software development practices. In this paper we present our approach for addressing those challenges related to ontology development support activities (i.e., documentation, evaluation, releasing and versioning). We describe OnToology, a web-based application that builds on top of git-based environments and eases the development process of ontologies in a collaborative manner. We have validated OnToology against a set of representative requirements for ontology development support activities in distributed environments. We also report on a survey of the system to assess its usefulness and usability.

Actions

Below we list actions we asked the users to perform for evaluation the usability of OnToology:
  1. Login and authorize OnToology application
  2. Add a repository to watch
  3. Authorize application to access public repository
  4. See watched repositories and check that yours is there.
  5. Check that OnToology has generated a pull request with the files
  6. Merge using GitHub the generated resources by OnToology
  7. Edit your ontology and commit it to GitHub
  8. Generate previsualization
  9. Change configuration using the web interface
  10. Force the generation of documentation, diagrams and evaluation
  11. Publish the ontology with w3id
  12. Check the published ontology with the w3id
  13. Download the bundle
  14. Stop watching the repo and check that it does not appear anymore
